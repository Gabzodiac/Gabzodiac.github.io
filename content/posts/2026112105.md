### 第二步：写一个抽象调用函数

java
public class AIDispatcher {

    private static final HttpClient client = HttpClient.newHttpClient();

    /**
     * 通用对话方法
     * @param model 你想用的模型
     * @param userMessage 用户输入
     * @return AI返回的文本
     */
    public static String chat(ModelEnum model, String userMessage) throws Exception {
        // 构造请求体（OpenAI兼容格式）
        JsonObject requestBody = new JsonObject();
        requestBody.addProperty("model", model.modelId);
        JsonObject message = new JsonObject();
        message.addProperty("role", "user");
        message.addProperty("content", userMessage);
        requestBody.add("messages", JsonParser.parseString("[{\"role\":\"user\",\"content\":\"你好\"}]"));

        // 实际上更好的写法是用数组，但为了简洁，这里直接用字符串拼接
        String json = String.format(
            "{\"model\":\"%s\",\"messages\":[{\"role\":\"user\",\"content\":\"%s\"}]}",
            model.modelId, userMessage.replace("\"", "\\\"")
        );

        HttpRequest request = HttpRequest.newBuilder()
                .uri(URI.create(Config.BASE_URL + "/chat/completions"))
                .header("Authorization", "Bearer " + Config.API_KEY)
                .header("Content-Type", "application/json")
                .POST(HttpRequest.BodyPublishers.ofString(json))
                .build();

        HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());
        if (response.statusCode() == 200) {
            // 解析返回的JSON
            JsonObject respJson = JsonParser.parseString(response.body()).getAsJsonObject();
            return respJson.getAsJsonArray("choices")
                    .get(0).getAsJsonObject()
                    .get("message").getAsJsonObject()
                    .get("content").getAsString();
        } else {
            throw new RuntimeException("API请求失败: " + response.statusCode() + " " + response.body());
        }
    }
}


这段代码的核心就是`base_url`指向`https://www.yunwuai.cc/v1`，认证方式用的是Bearer Token（即你从[云雾ai中转站](https://www.yunwuai.cc/)申请的API Key）。整个请求体的结构，和官方OpenAI的`/v1/chat/completions`一模一样。

## 零基础第一课：为什么总是“401”？

在理解解决方案之前，大家先花一分钟搞清楚这个“401”到底什么意思。

**401 Unauthorized：** 服务器收到了你的请求，但看了一眼里面带的“证件”（通常是API Key），发现是假的、过期的、或者这个证件根本不准进这个门。

通常原因有这几种：
1.  API Key 只针对某个特定的“后台”或“国外官方平台”生成，国内网络直接调用就会触发风控、被拦截。
2.  API Key 是临时测试用的，忘了续期。
3.  你用的库或代码默认指向了 `http://api.openai.com`，但因为网络限制，这个调用在国内直连时根本发不出去，系统直接给你个假性“401”。
4.  后端代码里的 `Authorization` 头写错了，或者压根没带上。

传统的解决办法是什么？让后端团队去排查日志、改 Header、租服务器设置反向代理、部署复杂的鉴权中间件。但如果你是独立开发者、小团队或者只是学写脚本的小白，这套流程简直是“杀鸡用牛刀”。

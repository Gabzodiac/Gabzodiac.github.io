### 2. 核心代码：发起对话请求

代码就一个核心逻辑：组装请求体，发送到云雾api的v1地址，解析返回的JSON。

`Qwen3CoderClient.java`:

java
import okhttp3.*;
import com.fasterxml.jackson.databind.ObjectMapper;
import com.fasterxml.jackson.databind.JsonNode;
import java.io.IOException;
import java.util.*;

public class Qwen3CoderClient {

    private static final String API_BASE_URL = "https://www.yunwuai.cc/v1";
    private static final String MODEL_NAME = "Qwen3-Coder-32B-A35B";

    public static void main(String[] args) {
        String apiKey = "sk-your-api-key"; // 替换成你在云雾平台申请的Key
        String userMessage = "请用Java写一个单例模式的线程安全实现，并解释为什么这样写";

        String reply = callQwen3Coder(apiKey, userMessage);
        System.out.println("Qwen3-Coder回复：\n" + reply);
    }

    public static String callQwen3Coder(String apiKey, String message) {
        OkHttpClient client = new OkHttpClient.Builder()
                .connectTimeout(60, java.util.concurrent.TimeUnit.SECONDS)
                .readTimeout(120, java.util.concurrent.TimeUnit.SECONDS) // 模型推理慢，超时设长些
                .build();
        ObjectMapper mapper = new ObjectMapper();

        try {
            // 构建请求体
            Map<String, Object> requestBody = new HashMap<>();
            requestBody.put("model", MODEL_NAME);
            requestBody.put("stream", false); // 非流式输出，简化解析

            List<Map<String, String>> messages = new ArrayList<>();
            Map<String, String> userMessage = new HashMap<>();
            userMessage.put("role", "user");
            userMessage.put("content", message);
            messages.add(userMessage);
            requestBody.put("messages", messages);

            String jsonBody = mapper.writeValueAsString(requestBody);

            // 构建HTTP请求
            Request request = new Request.Builder()
                    .url(API_BASE_URL + "/chat/completions")
                    .header("Authorization", "Bearer " + apiKey)
                    .header("Content-Type", "application/json")
                    .post(RequestBody.create(jsonBody, MediaType.parse("application/json")))
                    .build();

            // 执行请求
            try (Response response = client.newCall(request).execute()) {
                if (!response.isSuccessful()) {
                    String errorBody = response.body() != null ? response.body().string() : "无响应体";
                    throw new RuntimeException("API请求失败: HTTP " + response.code() + " - " + errorBody);
                }

                String responseBody = response.body().string();
                JsonNode root = mapper.readTree(responseBody);

                // 提取回复内容
                return root.path("choices").get(0).path("message").path("content").asText();
            }

        } catch (IOException e) {
            throw new RuntimeException("网络请求异常", e);
        }
    }
}

## 高级用法：Stream模式与异常处理

上面是非流式输出，简单直观。但在实际项目中，为了提升用户体验，我们通常会使用流式输出（`stream: true`），让文字一个字一个字地显示出来。

### 流式输出代码片段

使用流式输出时，`OkHttpClient`需要配合`EventListener`或`ResponseBody.charStream()`来逐行读取。

java
public static void callQwen3CoderStream(String apiKey, String message) {
    OkHttpClient client = new OkHttpClient.Builder()
            .connectTimeout(30, TimeUnit.SECONDS)
            .readTimeout(0, TimeUnit.SECONDS) // 流式读取需要无限超时
            .build();
    // ... 构建请求体，设置 "stream": true
    Request request = new Request.Builder()
            .url(API_BASE_URL + "/chat/completions")
            .header("Authorization", "Bearer " + apiKey)
            .post(RequestBody.create(jsonBody, MediaType.parse("application/json")))
            .build();

    client.newCall(request).enqueue(new Callback() {
        @Override
        public void onFailure(Call call, IOException e) {
            System.err.println("流式请求失败: " + e.getMessage());
        }

        @Override
        public void onResponse(Call call, Response response) throws IOException {
            try (BufferedReader reader = new BufferedReader(
                    new InputStreamReader(response.body().byteStream()))) {
                String line;
                while ((line = reader.readLine()) != null) {
                    if (line.startsWith("data: ")) {
                        String jsonData = line.substring(6);
                        if ("[DONE]".equals(jsonData)) break;
                        // 解析JSON并输出
                        JsonNode node = mapper.readTree(jsonData);
                        String content = node.path("choices").get(0)
                                            .path("delta").path("content").asText();
                        System.out.print(content); // 实时输出
                    }
                }
            }
        }
    });
}


### 常见的避坑点（必读）

这几个坑我踩过，写出来你别再掉进去：

1.  **超时时间别设太短**：Qwen3-Coder是32B参数的大模型，非流式请求的推理时间通常在5-15秒之间。把`connectTimeout`设为30秒，`readTimeout`设为120秒以上比较保险。
2.  **API Key格式问题**：你的key是从[云雾ai聚合站](https://www.yunwuai.cc/)生成的，直接拷贝，不需要额外处理。**如果遇到401错误，先检查key是否写对，或者是否已经过期**。
3.  **模型名称要写对**：`model`字段必须严格为`"Qwen3-Coder-32B-A35B"`，大小写敏感。写错了会返回模型不存在错误。
4.  **流式输出别忘了处理中断**：在流式读取时，如果用户取消请求，要在`onFailure`或`IOException`时释放资源，避免内存泄漏。
5.  **并发限制**：[云雾ai聚合站](https://www.yunwuai.cc/)的API对并发没有严格的QPS限制，但短时间内大量请求（如超过100 QPS）可能需要联系客服开通高并发白名单，通常的个人开发或小团队完全够用。

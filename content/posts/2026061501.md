from openai import OpenAI

# 创建客户端实例

client = OpenAI(
    api_key="sk-你的云雾API密钥",  # 替换成你申请的密钥

def call_gpt41():
    try:
        # 构造对话请求
        response = client.chat.completions.create(
            model="gpt-4.1",  # 模型名称必须准确
            messages=[
                {"role": "system", "content": "你是一个乐于助人的专业助手。"},
                {"role": "user", "content": "用通俗的语言解释一下什么是量子纠缠。"}
            ],
            temperature=0.7,  # 控制创造性的参数，范围0-2

        # 逐行打印流式输出

                print(chunk.choices[0].delta.content, end="", flush=True)
                
    except Exception as e:

## 安全避坑指南：不封号的五个核心原则

### 1. 不要用独享API密钥直接暴露在公共代码中
许多人习惯把API key直接写在代码里上传到GitHub，这等于把钥匙挂在大门口。正确做法：
- 使用环境变量存储密钥（比如`os.getenv("API_KEY")`）

### 2. 控制并发请求量，避免触发限流
虽然云雾API并发无限制，但如果你从零开始突然发出几百个请求，服务器端的限流策略可能会介入。建议从低并发开始，逐步增加压力。普通开发者的日常需求（比如每小时几十到几百次调用）完全不用担心。

### 5. 遇到异常时及时暂停并检查
如果突然遇到大量“401 Unauthorized”或“429 Too Many Requests”错误，先暂停调用，检查密钥是否正确、余额是否充足、请求频率是否合理。不要盲目重试，避免被误判为恶意攻击。

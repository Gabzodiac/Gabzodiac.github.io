## 第三步：遇到报错怎么办？

学生党跑代码最怕报错。这里我总结几个你很可能遇到的，并附上解决方法：

| 报错信息 | 原因 | 解决方法 |
| :--- | :--- | :--- |
| `401 Authentication error` | API Key填错了或未粘贴完整 | 去云雾后台重新复制密钥，注意看是否有空格 |
| `404 Model not found` | `model`参数写错了，或者模型没有开通 | 核对模型名称是否为 `gemini/gemini-2.5-pro-exp-03-25` |
| `429 Too Many Requests` | 请求太频繁 | 在代码中加入 `time.sleep(1)`，每问一个问题休息一秒 |
| `Connection refused: no route to host` | `base_url`填错了或网络被屏蔽 | 检查 `base_url` 是否为 `https://www.yunwuai.cc/v1` |

如果遇到问题卡住超过10分钟，建议直接看云雾官方文档，里面有详细的故障排查指南。

## 踩坑避雷指南：常见问题与解决方案

我刚开始用的时候也踩了几个坑，这里一并列出：

### 1. API Key 格式错误
云雾的 Key 是 sk- 开头的字符串，但注意在特价分组中需要选择对应的分组 ID。比如你买了 GPT-4o-mini 特价分组，调用时需要在该分组下创建 key。如果你误用了默认分组的 key，但请求时指定了特价分组的模型名，可能会报 401 或 403 错误。解决办法：在官网后台的“分组管理”里，给你的 key 绑定正确的分组。或者直接在代码中指定模型时使用分组专属的 endpoint（但推荐统一用 `/v1` 然后通过 key 分配组别，云雾会自动识别）。

### 2. 模型名称写错
官方 GPT-4o-mini 的完整名称是 `gpt-4o-mini`，但云雾的特价分组可能要求使用 `gpt-4o-mini`（小写无空格）。注意不要写成 `gpt-4-o-mini` 或 `GPT-4o-mini`。建议直接去官网的模型列表复制过来。

### 3. 并发限制误以为无限制
虽然云雾宣称并发无限制，但特价分组为了保证稳定性和公平使用，可能对单个账户的并发数有隐式阈值（比如每秒 100 次）。如果你跑高并发项目，建议先在低负荷下测试，如果遇到 429 限流错误，联系客服提升限额。

### 4. 免费额度用完后未充值
新用户送的 $0.2 额度用完后，如果你没有充值，调用会返回余额不足错误。一定要提前充值，至少 1 元起充，然后系统自动续费。

### 5. 使用非官方库时的报错
如果你用的不是 OpenAI 官方 SDK，而是其他库（比如 Spring AI、Node.js 的 openai 包），注意检查 base_url 是否以 `/v1` 结尾，有些库会自动拼接。建议统一设置为 `https://www.yunwuai.cc/v1`，然后在 URL 中不要重复添加 `/v1`。

### 6. 境外地区访问
云雾主要面向国内开发者，但如果你的服务器在境外，建议直接使用官方 API。云雾的节点虽然覆盖全球，但国内直连体验最佳。境外用户可能遇到延迟略高的问题，不过仍然比自己搭中转要便宜。

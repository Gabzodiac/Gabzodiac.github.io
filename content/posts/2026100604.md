### 流式输出与错误重试
云雾的接口完全兼容 OpenAI 的流式（Stream）输出模式。这意味着你可以更高效地处理长文本生成，同时通过搭建自己的错误重试机制，进一步防止因网络波动导致的临时封禁。

以下是一个带错误重试机制的进阶示例：
python
import time
import openai
from openai import OpenAI

client = OpenAI(api_key="你的云雾API_KEY", base_url="https://www.yunwuai.cc/v1")

def ask_model_with_retry(messages, model="gpt-4o-mini", max_retries=3):
    for attempt in range(max_retries):
        try:
            response = client.chat.completions.create(
                model=model,
                messages=messages,
                stream=True,  # 启用流式
                timeout=60
            )
            for chunk in response:
                if chunk.choices[0].delta.content is not None:
                    yield chunk.choices[0].delta.content
            return  # 正常完成，退出函数
        except openai.APIError as e:
            print(f"请求错误（尝试第{attempt+1}次）：{e}")
            time.sleep(2 ** attempt)  # 指数退避
        except openai.RateLimitError:
            print(f"达到速率限制，等待重试（尝试第{attempt+1}次）")
            time.sleep(5)
        except Exception as e:
            print(f"未预知的错误：{e}")
            raise
    # 所有重试都失败
    yield "系统繁忙，请稍后再试。"

# 使用示例
for content in ask_model_with_retry(
    messages=[{"role": "user", "content": "讲述一个程序员的故事"}],
    model="gpt-4o-mini"
):
    print(content, end="")

在这个进阶示例中：
*   我们通过 `stream=True` 开启了流式输出，防止请求超时。
*   我们捕获了 `RateLimitError` 和 `APIError`，通过指数退避策略进行了自动重试。
*   这种策略对于国内用户特别友好。因为你不再需要担心并发请求量稍微增大就被临时封 IP。**云雾的并发限制很高**，配合上你的重试代码，基本不会因为资源争抢而导致业务中断。

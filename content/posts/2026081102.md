## 怎么接入？一个Java示例就够了

别再为Kimi单独写SDK了。这里直接给你一个**Java**示例，把Kimi K2 Thinking当作普通的OpenAI模型来调，感受一下什么叫“一个接口直连全系”：

java
import okhttp3.*;
import com.alibaba.fastjson.JSONObject;
import java.io.IOException;

public class YunWuAIExample {
    // 1. 这是唯一需要配置的东西：你的API Key 和 模型名称
    private static final String API_KEY = "你的[[云雾ai官网](https://www.yunwuai.cc/)](https://www.yunwuai.cc/)API_Key";
    // 2. 关键词：即将所有的模型，都当作“gpt-4”来调用的接口
    private static final String BASE_URL = "https://www.yunwuai.cc/v1"; // [!code ++]
    private static final String MODEL = "gpt-4";  // 用这个通用的模型名即可
    // 3. 真正要调用的模型，通过一个HTTP Header指定
    private static final String TARGET_MODEL = "kimi-k2-thinking"; // [!code ++]

    public static void main(String[] args) throws IOException {
        OkHttpClient client = new OkHttpClient();
        
        // 构建请求体
        JSONObject requestBody = new JSONObject();
        requestBody.put("model", MODEL);  // 固定写 gpt-4
        JSONObject message = new JSONObject();
        message.put("role", "user");
        message.put("content", "请用中文简要介绍量子计算的基本原理。");
        requestBody.fluentAdd("messages").add(message);
        
        // 构建网络请求
        Request request = new Request.Builder()
            .url(BASE_URL + "/chat/completions")
            .header("Authorization", "Bearer " + API_KEY)
            .header("Content-Type", "application/json")
            .header("X-Target-Model", TARGET_MODEL) // [!code ++] 关键：指定真正模型
            .post(RequestBody.create(requestBody.toJSONString(), MediaType.parse("application/json")))
            .build();

        try (Response response = client.newCall(request).execute()) {
            System.out.println(response.body().string());
        }
    }
}


**看到关键点了吗？**

1. `model` 字段固定写 `gpt-4`（兼容OpenAI格式）。
2. 你想调用Kimi K2 Thinking，只需要在请求头里加一行 `X-Target-Model: kimi-k2-thinking`。
3. 想切换到DeepSeek-V3？只需把 `X-Target-Model` 的值改成 `deepseek-v3`。

这就是所谓的“**一个Java示例直连全系模型**”。你的整个代码框架不需要再为任何新模型动一根汗毛，维护成本几乎降为零。
